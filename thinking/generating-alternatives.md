# 代替案生成（緑ハット）

発散に特化。収束は `evaluating-risk-and-opportunity.md` に任せる。

## 手順

### Step 1: 現在の選択肢を書き出す
各選択肢を「動詞 + 目的語」で。曖昧表現禁止。

### Step 2: 暗黙の前提を洗い出す
- なぜこの2つだけが選択肢なのか？
- 誰がこの制約を決めたか？本当に動かせないか？
- 初めてこの問題を見た人ならどう考えるか？

### Step 3: 前提を1つずつ外す
「もし○○が不要だったら？」「逆だったら？」
**この段階では「現実的かどうか」を判断しない。**

### Step 4: 制約を変形する
| 変形 | 問い |
|------|------|
| スケール変更 | 10倍/10分の1だったら？ |
| 時間軸変更 | 期限が倍/半分だったら？ |
| リソース変更 | 1人で作るなら？ |
| 優先度逆転 | 最優先と最低優先を入れ替えたら？ |
| 時代変更 | 3年後に判断するなら？ |

### Step 5: 組み合わせる
| 型 | 方法 |
|---|------|
| 良いとこ取り | AとBの長所を合わせる |
| 時間分割 | Aを先にやってBに移行 |
| 空間分割 | 領域でA/Bを使い分け |
| 段階的移行 | 新規はB、既存はA |
| プロトタイプ検証 | 両方を小さく試して比較 |

### Step 6: 軽量評価（黒1つ・黄1つで有望な案を残す）

## ルール
1. 発散フェーズでは「現実的か」を判断しない。「それは無理」禁止
2. 最低3つの代替案を出してから評価
3. 極端な案を1つは必ず含める
4. 前提を言語化してから外す
5. 発散と収束を混ぜない
