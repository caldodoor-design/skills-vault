# 完了前の検証

**鉄則: 新鮮な検証証拠なしに完了を主張してはならない。**

## ゲート関数
1. **特定**: この主張を証明するコマンドは何か？
2. **実行**: 完全なコマンドを実行（新規、完全に）
3. **読解**: 全出力を読み、終了コード確認、失敗数カウント
4. **検証**: 出力が主張を裏付けているか？
5. **報告**: 証拠付きで報告

## 必要な証拠
| 主張 | 必要な証拠 |
|------|-----------|
| テスト通過 | テストコマンド出力: 0件の失敗 |
| ビルド成功 | ビルドコマンド: exit 0 |
| バグ修正済み | 元の症状のテスト通過 |
| 回帰テスト | Red-Greenサイクル検証済み |
| エージェント完了 | VCS差分で変更を独立確認 |

## 危険信号（停止）
「はず」「おそらく」を使っている、検証前に「完了！」と表明、エージェントの報告を鵜呑み、「今回だけは」

**検証に近道はない。コマンド実行 → 出力読解 → 結果主張。**
