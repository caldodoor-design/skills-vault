# リサーチ付きコンテンツ執筆支援

## ワークフロー
1. **理解**: トピック・主張・読者・長さ・目的・文体を確認
2. **アウトライン**: フック→導入→本文セクション→結論→リサーチToDo
3. **リサーチ**: 知見を引用付きで整理（ソース検証、最新データ優先）
4. **フック改善**: データ駆動型/質問型/ストーリー型の3案を提示
5. **セクション別FB**: 明瞭性・流れ・根拠・スタイルの4軸でレビュー
6. **最終レビュー**: 構造・コンテンツ品質・技術品質を総合評価

## 文体保持の原則
- スタイルを学ぶ、提案する（置き換えない）、トーンを合わせる
- 定期確認:「これはあなたらしい表現ですか？」

## コンテンツ種別ガイド
| 種別 | フロー |
|------|--------|
| ブログ | アウトライン→リサーチ→導入+FB→本文+FB→結論→推敲 |
| ニュースレター | フック→簡易アウトライン→一気に下書き→仕上げ |
| 技術チュートリアル | 手順アウトライン→コード例→説明→テスト→精度確認 |
| ソートリーダーシップ | 独自の切り口→既存視点リサーチ→テーゼ→強いPOVで執筆 |
