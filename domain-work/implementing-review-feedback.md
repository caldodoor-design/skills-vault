# レビューフィードバック実装

## ワークフロー
1. **解析**: フィードバック項目を分割・抽出（曖昧な点は着手前に確認）
2. **Todoリスト化**: 各項目を具体的・測定可能なタスクに変換
3. **実装**: Grep/Globで対象コード特定→Edit→検証→Todo更新（in_progressは1つだけ）
4. **バリデーション**: テスト実行、lint確認、既存機能の維持確認
5. **報告**: 変更サマリーを提示

## フィードバック種別
| 種別 | 対応 |
|------|------|
| コード変更 | Edit、型ヒント規約（PEP 604/585）に従う |
| 新機能 | Write + テスト追加 + ドキュメント更新 |
| テスト | 関数として記述、pytest規約 |
| リファクタリング | 機能維持、テストでリグレッション確認 |

## エッジケース
- **矛盾するFB**: ユーザーに確認、矛盾を明確に説明
- **破壊的変更**: 実装前に通知、影響と代替案を議論
- **テスト失敗**: Todo完了前にテスト修正
- **参照コードなし**: ユーザーに確認
