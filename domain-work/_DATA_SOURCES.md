# Data Sources (for Verification)

このプロジェクトで扱う「検証データ」の方針。
※実データは機密になり得るため、共有範囲に注意。

---

## ✅ データ分類
- Sample A：列が綺麗な通帳（ヘッダあり・行番号あり）
- Sample B：列が詰まった通帳（摘要が長い）
- Sample C：歪み・影・手書きが混在する通帳（難易度高）

---

## ✅ Sprint 3 に必要な検証データ
- bankId=default（未登録想定）
- bankId=任意の登録済み銀行（1つでOK）
- headerStrategy=IGNORE を想定したケース（強制失敗）

---

## ✅ ルール
- 画像・PDFは GitHub に上げない（必要ならローカルで管理）
- 共有する場合は「匿名化済みのダミー」だけ
- 検証は “現象” と “再現条件” をレポートに残す
