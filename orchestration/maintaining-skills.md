# Skillsメンテナンス

定期的にSkillsの品質を維持するためのチェックリスト。

## チェック項目

- [ ] 500行を超えているSkillはないか → 超えていたらreference/*.mdに分割
- [ ] 重複した内容を持つSkillはないか → 統合を検討
- [ ] 3ヶ月以上更新されていないSkillはないか → 廃止を検討
- [ ] Claudeが最初から知っている内容を説明していないか → 削除
- [ ] 時間依存の情報が含まれていないか → 削除または更新
- [ ] descriptionは「何をするか」＋「いつ使うか」が明確か
- [ ] 1 Skill = 1フェーズ = 1ブロックになっているか → 複数目的なら分割
- [ ] デフォルト値が決まっていない選択肢が並んでいないか

## 実行手順

1. `.claude/skills/` 配下の全ファイルを一覧取得
2. 各ファイルの行数をチェック（500行超過の検知）
3. 各ファイルのYAMLフロントマターを確認（description品質チェック）
4. 内容の重複を目視チェック
5. 問題があればAskUserQuestionで報告・判断を仰ぐ
