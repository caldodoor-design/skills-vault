# 必須ワークフロー

## 基本フロー
1. **計画確認**: 指示された計画・DoDを確認
2. **実装**: コード修正
3. **検証**: ビルド・テスト実行
4. **レビュー**: Codexレビュー
5. **報告**: 結果報告

## 堅牢なワークフロー（フリーズ回避）
各ステップを**分割実行**。「修正→ビルド→レビュー」の一括指示は避ける。

## 報告フォーマット
```
### 実施内容
### 変更ファイル
### 検証結果（コマンドと結果）
### DoD達成状況
### 備考
```
